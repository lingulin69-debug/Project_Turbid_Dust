# 美術素材規格書 (Asset Specifications)

為了確保網頁在手機端 (尤其是中國網路環境) 能流暢運行並呈現最佳視覺效果，請依照以下規格準備素材。

## 1. 檔案路徑結構
請將圖片檔案存放在專案的 `public/assets/` 目錄下 (若目錄不存在請建立)。

```
public/
  assets/
    map/
      world_map_bg.webp    (大地圖底圖)
    icons/
      town.png             (城鎮圖示)
      hospital.png         (醫院圖示)
      church.png           (教堂圖示)
      default.png          (預設據點圖示)
    ui/
      scale_base.png       (天秤底座)
      scale_arm.png        (天秤擺動臂)
```

## 2. 詳細規格要求

### A. 世界地圖 (World Map)
*   **檔名**：`world_map_bg.webp` (或 `.jpg`)
*   **推薦尺寸**：寬度 **3000px ~ 4500px** (視地形細節而定)
*   **檔案格式**：
    *   **首選**：**WebP** (壓縮率高，畫質好)
    *   **備選**：JPG (品質 80-85%)
    *   **禁止**：PNG (除非有透明背景需求，否則檔案會過大)
*   **檔案大小限制**：**< 3MB** (強烈建議，避免載入過久)
*   **繪製注意**：請繪製純地形與道路，**不要**將 UI 圖示 (如城鎮建築) 畫死在底圖上，這樣程式才能製作互動效果。

### B. 據點 UI 圖示 (Landmark Icons)
*   **檔名**：`town.png`, `hospital.png`, `church.png` 等
*   **推薦尺寸**：**128x128px** 或 **256x256px** (Retina 螢幕適用)
*   **檔案格式**：**PNG** (必須支援透明背景)
*   **繪製注意**：
    *   建議在圖示底部留一點空白或陰影，讓它疊加在地圖圓點上方時有「懸浮感」。
    *   風格請統一 (例如都有描邊或都是平面化)。

### C. 中央天秤 (Balance Scale)
這是畫面的視覺核心，需要拆成兩層以實作動畫。

#### 1. 底座 (Base)
*   **檔名**：`scale_base.png`
*   **描述**：固定不動的支架部分。
*   **尺寸**：寬度約 **300-500px**。

#### 2. 擺動臂 (Arm/Pan)
*   **檔名**：`scale_arm.png`
*   **描述**：會左右傾斜的橫桿與盤子。
*   **重要**：請確保圖片的 **幾何中心點 (Center)** 就是 **旋轉軸心 (Pivot Point)**。如果重心偏了，旋轉時會看起來很奇怪。
*   **尺寸**：配合底座比例。

## 3. 測試方式
1.  將圖片放入對應資料夾。
2.  重新整理網頁。
3.  若圖片未出現，請檢查檔名是否完全一致 (大小寫敏感)。
