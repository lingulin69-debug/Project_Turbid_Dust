## 讀這份文件前，先了解三件事

### 📁 什麼是 `public/` 資料夾？
專案裡有一個叫 `public/` 的資料夾。
**放進這個資料夾的圖片，網頁可以直接抓到。**
你不需要懂程式，只要把圖檔放對位置、取對名字，程式就會自動顯示。

### ⚠️ 檔名大小寫有差！
`Portrait_Unknown.png` 和 `portrait_unknown.png` 是**不同的檔案**。
**請全部用小寫英文字母 + 底線命名。**

### ✅ 測試方法
圖片放好後，重新整理網頁。
如果圖片沒出現，99% 是**檔名拼錯**或**放錯資料夾**。

## 一、角色立繪 (`public/assets/portraits/`)

立繪會在玩家**點開據點視窗**時，出現在視窗右側。
手機版不顯示（自動隱藏），桌機版才看得到。

### 尺寸規格

| 項目 | 規格 |
|------|------|
| **顯示尺寸** | 寬 560px × 高 800px |
| **建議製作尺寸** | 寬 1120px × 高 1600px（@2x，高清螢幕用） |
| **格式** | PNG（建議去背；也可以有背景，視風格而定） |
| **顯示方式** | 圖片**頂部對齊**，填滿寬度（超出高度的部分會被裁掉） |

> 💡 **建議**：角色頭部放在圖片上方 1/4 處，這樣即使被裁切也能看到臉。

### 檔案命名規則

| 檔名 | 用途 |
|------|------|
| `landmark_l1_t01.png` | 據點 l1_t01 的專屬立繪 |
| `landmark_l1_t02.png` | 據點 l1_t02 的專屬立繪 |
| `landmark_l1_p01.png` | 據點 l1_p01 的專屬立繪 |
| `portrait_unknown.png` | **通用替代圖**（沒有專屬立繪時自動顯示） |

> 📌 **據點 ID 對照**：請見地圖或詢問程式，每個據點都有一個像 `l1_t01` 的 ID。
> 據點立繪的檔名就是 `landmark_` + 據點 ID + `.png`。

> 💡 **建議工作流程**：
> 1. 先把 `portrait_unknown.png` 換成你想要的「通用背景圖」
> 2. 之後再慢慢補上各據點的專屬立繪
> 3. 每放一張，重新整理網頁確認有顯示

## 二、NPC 互動視窗底圖 (`public/assets/npc-bg/`)

玩家點擊地圖上的 NPC（商人、旅店老闆等）時，會跳出互動視窗。
這張底圖就是視窗的**全版背景**，可以包含角色立繪，兩者合為一張圖。

### 設計概念

```
┌─────────────────────────────┐
│  ← 這整張就是一張圖          │
│                             │
│   商店場景 + 角色立繪         │
│   全部畫在同一張上            │
│                    [角色]    │
│                             │
└─────────────────────────────┘
  文字 UI（商品列表、按鈕）
  由程式疊加在底圖上方
```

### 尺寸規格

| 項目 | 規格 |
|------|------|
| **視窗最大寬度** | 420px |
| **建議製作寬度** | 840px（@2x 高清） |
| **建議製作高度** | 600 ~ 900px（依 NPC 種類，高度不固定） |
| **格式** | PNG 或 JPG |
| **注意** | 圖片下半部會被 UI 文字覆蓋，重要角色畫面建議放在上半部 |

### 檔案命名規則

| 檔名 | NPC |
|------|-----|
| `npc_black_merchant_bg.png` | 🎭 黑心商人 |
| `npc_item_merchant_bg.png` | 📦 道具商人 |
| `npc_inn_owner_bg.png` | 🏠 旅店老闆 |
| `npc_pet_merchant_bg.png` | 🐾 寵物商人 |
| `npc_trafficker_bg.png` | 🪤 人販子 |

> 💡 **建議**：先做一張，放進去看效果，確認滿意再做其他的。

## 三、世界地圖底圖 (`public/assets/map/`)

遊戲的主畫面地圖背景。

### 尺寸規格

| 項目 | 規格 |
|------|------|
| **建議製作寬度** | 3000 ~ 4500px |
| **格式** | **WebP**（首選）或 JPG（品質 80–85%）|
| **禁止格式** | PNG（檔案太大，載入慢）|
| **檔案大小限制** | **< 3MB**（超過會讓手機使用者等很久）|

### 重要注意事項

> ❌ **不要**把 UI 元件（城鎮圖示、角色圖示）畫在地圖底圖上。
> ✅ 底圖只畫地形、道路、地貌。
> 城鎮位置由程式另外疊加，這樣才能做點擊互動。

### 檔案命名規則

| 檔名 | 用途 |
|------|------|
| `world_map_bg.webp` | 主世界地圖 |

## 四、據點地圖小圖示 (`public/assets/icons/`)

### 尺寸規格

| 項目 | 規格 |
|------|------|
| **建議尺寸** | 128 × 128px 或 256 × 256px |
| **格式** | PNG（必須去背） |

### 檔案命名規則

| 檔名 | 據點類型 |
|------|---------|
| `town.png` | 城鎮 (town) |
| `church.png` | 教堂 / 聖地 (church) |
| `school.png` | 學校 / 圖書館 (school) |
| `hospital.png` | 醫院 (hospital) |
| `default.png` | 其他未知類型 |

## 五、UI 裝飾元素 (`public/assets/ui/`)

畫面中的特殊 UI 元件（目前主要是天秤）。

### 天秤 — 需拆成兩層

天秤動畫需要兩張圖分開，程式才能讓它傾斜：

| 檔名 | 描述 | 尺寸 |
|------|------|------|
| `scale_base.png` | 固定不動的底座支架 | 約 300–500px 寬 |
| `scale_arm.png` | 會左右傾斜的橫桿 | 配合底座比例 |

> ⚠️ **`scale_arm.png` 的旋轉中心點很重要！**
> 橫桿的**幾何中心**（圖片正中間）必須是旋轉的軸心。
> 如果畫的圖偏左或偏右，旋轉時會看起來怪怪的。
> 建議：先畫好橫桿，再把整張畫布調整成以橫桿中心為中心點的正方形。

## 常見問題

**Q：圖片放好了但網頁上沒出現？**
A：99% 是檔名問題。請確認：
1. 全小寫了嗎？（`portrait_unknown.png` 不是 `Portrait_Unknown.png`）
2. 副檔名對嗎？（`.png` 不是 `.PNG`）
3. 放對資料夾了嗎？

**Q：圖片出現但很模糊？**
A：你製作的尺寸太小。請依照「建議製作尺寸」（@2x）製作。

**Q：圖片出現但被裁掉了？**
A：正常現象。請調整圖片構圖，把最重要的部分放在建議位置。

**Q：我要怎麼知道 `l1_t01` 是哪個據點？**
A：請問程式或查看地圖，每個據點都有對應的 ID。
